Java 자료형

정수형: byte(1)<short(2)<int(4)<long(8) / char(2) 문자도 정수형으로 취급, 주어진 번호가 있음 (자바의 경우 Unicode) 단, char은 양의 정수만을 저장할 수 있음 → short와 표현 가능한 수는 동일하나 범위가 다름 실수형: float < double 정수형은 용량이 클수록 더 큰 수를 저장할 수 있고, 실수형은 용량이 클수록 더 정확한 수를 저장할 수 있다 (지수 / 가수 ← 의미 있는 수를 저장) * long형은 뒤에 대문자 L * double, float형은 뒤에 대문자 또는 소문자 사용 가능 문자는 정수형이지만 문자열은 객체형에 해당함. byte, short, char는 계산 시 int로 변환이 필요하므로 계산이 필요한 경우 int를 사용하는 것이 빠름. 💡자료형이 다른 경우 비교 연산? 네, Java에서는 **자료형이 다르더라도 비교 연산(`>`, `<`, `>=`, `<=`, `==`, `!=`)이 가능**해요! ✅ 이유: Java의 자동 형변환 (Implicit Type Conversion) Java에서는 서로 다른 자료형을 비교할 때 **더 큰 자료형으로 자동 변환 (promotion)**이 일어납니다. 예: `byte`형 변수 `b`는 `int`로 변환된 후 비교됩니다.
